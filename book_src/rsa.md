# RSA 混合加/解密與簽章

傳統的 RSA 非對稱加密演算法由於數學限制，無法直接加密大於金鑰長度（如 2048-bit 只能加密少於 245 位元組）的資料。

Kitana 採用 **Hybrid Encryption (混合加密)** 機制，將 **RSA 的金鑰交換便利性** 與 **AES 的大檔案處理能力** 結合，徹底克服了此限制。

## 核心機制

### 混合加密 (Hybrid Encrypt)
1. 系統自動在記憶體中生成一組隨機的一次性 **AES-256 金鑰與 IV**。
2. 用這組隨機 AES 金鑰將您的目標檔案/文字進行對稱加密。
3. 使用接收方的 **RSA 公鑰 (Public Key)** 加密這組 AES 金鑰與 IV。
4. 將加密後的金鑰長度（2 節點頭部碼）、加密金鑰包（256 位元組）與加密檔案內容依序拼接成一個 `.enc` 檔案。

### 混合解密 (Hybrid Decrypt)
1. 讀取密文頭部的 RSA 密鑰區塊。
2. 用您持有的 **RSA 私鑰 (Private Key)** 解密，還原出 AES 金鑰與 IV。
3. 用該 AES 金鑰解密後面所有的密文內容，還原出原始檔案或文字。

---

## 操作步驟

### 1. 混合加密 (Hybrid Encrypt)
1. 切換至 **RSA Crypt** 分頁，選擇模式為 **Hybrid Encrypt (AES+RSA)**。
2. 在 **Key File** 載入接收方的 **RSA 公鑰** (`.pub` 或 `.pem` 格式)。
3. 在 **Text / File** 輸入要加密的文字，或點擊瀏覽（或直接拖放）選擇要加密的檔案。
4. 點擊 **Execute**。加密後的密文或檔案將自動生成。

### 2. 混合解密 (Hybrid Decrypt)
1. 選擇模式為 **Hybrid Decrypt (AES+RSA)**。
2. 載入您自己的 **RSA 私鑰**。
3. 輸入/拖放要解密的密文或加密檔案。
4. 點擊 **Execute** 解密。

### 3. 數位簽章 (Sign & Verify)
* **Sign (簽章)**：使用您的**私鑰**對輸入的內容進行簽章。產生的簽章（Signature）會顯示在 Result 框中，可用於證明該資料是由您發送且未被篡改。
* **Verify (驗證)**：載入發送方的**公鑰**，輸入原始資料與對應的簽章密文，點擊執行。若通過驗證，系統會提示 `Signature Valid!`。
